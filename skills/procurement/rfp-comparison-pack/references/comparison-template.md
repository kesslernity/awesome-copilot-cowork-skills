# Comparison Pack — Structure & Rules

The structure for the DRAFT comparison pack. The pack supports a panel's evaluation; it never scores, ranks, or recommends.

## Document structure

1. **Header** — DRAFT line: "DRAFT supplier comparison for <category>, generated <date>. For panel evaluation only — not a score, ranking, or recommendation." Plus: category, number of suppliers, number of requirements, date.
2. **Comparison matrix** — see below.
3. **Gaps by supplier** — per supplier, requirements not addressed or unclear.
4. **Clarification questions** — per supplier, specific questions for the panel to raise.
5. **Neutral observations** — factual differences only.

## Matrix rules

| Requirement | Supplier A | Supplier B | Supplier C |
|---|---|---|---|
| [requirement 1] | [quote/cite or "Not addressed"] | ... | ... |

- One row per requirement (use the requirements list order).
- One column per supplier.
- Each cell: a short quote or a clause/section citation from that supplier's response, or "Not addressed".
- Never infer an offer that is not stated. Never write "best", "cheapest", "compliant", "preferred", or any score.
- Put prices, if present, in their own factual row — no "cheapest"/"best value" labels.

## What to never include

- Scores, weights, ratings, RAG status, rankings.
- A recommended or preferred supplier.
- Compliance pass/fail determinations.
- Any view on which response is better.

All of those are the evaluation panel's decisions, made under the organisation's procurement governance.
