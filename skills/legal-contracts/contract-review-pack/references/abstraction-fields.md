# Key-Terms Abstraction Fields

The fields to extract for the Key-Terms Abstraction table. For each, record the **value** and the **clause/section reference**. If a field is not present, record "Not located — verify". Never infer or invent.

## Core fields (extract for every contract)

| Field | What to capture |
|---|---|
| Parties | Full legal names and signing entities |
| Contract type | NDA / MSA / SOW / licence / services / other |
| Effective date | Commencement date or trigger |
| Term | Fixed term, rolling, or perpetual |
| Renewal | Auto-renew? Notice to prevent renewal? |
| Termination | Rights, notice periods, for cause vs convenience |
| Value / fees | Price, fee structure, payment terms |
| Limitation of liability | Cap amount/basis, mutual or one-sided, carve-outs |
| Indemnities | Who indemnifies whom, scope, caps |
| Intellectual property | Ownership, licences granted, background vs foreground IP |
| Confidentiality | Obligations, term, survival |
| Data protection | Controller/processor roles, DPA reference, transfers |
| Warranties | Given by each party, disclaimers |
| Governing law | Stated law — REQUIRED; flag prominently if absent |
| Dispute resolution | Courts / arbitration / venue / escalation |
| Assignment | Restrictions, change-of-control |

## Conditional fields (extract if the contract type calls for them)

- Service levels / SLAs and remedies (services/MSA)
- Acceptance criteria and process (SOW)
- Insurance requirements
- Force majeure
- Non-solicitation / non-compete
- Audit rights
- Exit / transition assistance

## Generic reviewer checklist (use when no playbook is supplied)

When the user has not supplied a playbook, build the Deviations & Issues section from these neutral checks instead — phrased as points for a reviewer to consider, never as advice:

- Is the limitation of liability mutual, and are the usual carve-outs present?
- Are indemnities reciprocal and capped, or open-ended?
- Is IP ownership unambiguous, and are licences scoped?
- Is the termination-for-convenience right balanced between the parties?
- Are auto-renewal and notice periods clearly stated and diarised?
- Is governing law stated and consistent with the dispute-resolution clause?
- If personal data is processed, is there a DPA or data-protection clause?
- Are there any defined terms used but not defined, or internal cross-reference errors?

Every item above is a point to raise for human review, not a determination.
